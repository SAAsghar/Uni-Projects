RAD Approach

The fastest approach of the 5

The RAD Approach stands for Rapid Application Development and focuses mainly on being able to release a single version of software as fast as possible whilst meeting all set requirements and maintaining reliability, accuracy and consistency for small to medium sized projects. This approach doesn't follow any set procedure to develop software, it can be achieved through any order of the software development cycle; lacks any formal stages.

This is the only approach in which the reuse of code and modules is encouraged for a quicker production. CASE tools are used throughout to test, design, plan and add elements to the program whilst developing th program itself. Due to the low costs, this approach is very good for small budget projects.

Advantages:

- Reuses sections of code and modules

- Low budget

- Professional programmers involved

Disadvantages:

- Lack of formal stages

- High reliance on CASE tools
