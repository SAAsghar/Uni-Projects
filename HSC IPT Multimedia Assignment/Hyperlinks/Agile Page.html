Agile Approach

Releases multiple working versions quickly

The agile approach focuses mostly on team development and is best suited for developing programs which require constant updates, such as websites. This approach doesn't follow a set flow of processes prior to completion unlike the structured approach. This approach aims to release multiple working versions quickly through a high communication with the user. Multiple versions are released, each with minor changes made throughout with a new iteration released for every set of changes made, this process continues throughout the life of the program.

This is the best approach to use when the final product will undergo further updates in the future as it adapts easily to changes made by the developer after release.

Advantages of using this approach are:

- Releases working versions quickly

- High communication with the user

- Adaptable to changes

- Professional programmers involved

Disadvantages of this approach are:

- Multiple versions are released
